Registered Logical Link Extension Point Documentation

Abstract [Index]

This package supports the extension point com.ibm.ccl.soa.deploy.core.logicalLink. It provides the framework by which logical links defined using the extension point are implemented in the model. In this sense a logical link describes a high level relationship between two DeployModelObjects. That is, it defines a constraint. Given a logical link, the framework uses a link tester to determine which validator to use to determine if the intent of the logical link has been implemented in the topology. If the intent has not been implemented, a number of implementations may be available to implement it. Registered logical link support includes:

  • A registry of logical links defined using the extension point.
  • A matcher for determining whether or not one of the registered logical links can be created between any two DeployModelObjects.
  • A validator used to determine whether or not the intent of a logical link is implemented in the topology model.
  • A resolution generator used to determine whether or not a predefined implementation of an unimplemented link exists.
