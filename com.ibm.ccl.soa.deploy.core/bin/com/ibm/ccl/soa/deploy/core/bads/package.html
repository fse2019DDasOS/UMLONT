The SOA Deploy Business Application Topology decorator semantic of the SOA Deployment Tooling allows architects to customize the tooling for business architect's user experience.

SOA Deploy Business Application Topology Decorator Semantic

SOA Deployment Business Application Topology Domain [Index]

 

The SOA Deployment Business Application Topology (BADS) is an extension domain of the SOA Deployment Editor which will allow Deployment Architects to abstractly layout their business components for deployment. This document will focus on the implementation of the decorator semantic that will customize the user experience for the business architect's usage. The types and the visual artifacts will be filtered and/or added, based on the needs of the business application topology developer.

 

Business Application Decorator Semantic Overview [Top]

 

The business application topology decorator semantic (BADS)is aimed at easing the user experience for the business application topology development. The core decorator semantic framework is used to provide an extension of business decorator semantic. The domain types and the UI views, pallette entries are filtered along by providing simple contributions.

 

This design will address the following key requirements of a business application decorator semantic contribution:

  • Extension to contribute a BADS
  • EMF Type enablement extension for BADS
  • UI Type enablement extension for BADS
    • Enabled Views/Menus/Pallette for BADS and their content
  • Extended validation for BADS

 

Note: The decorator demantic design  is used as a reference (com.ibm.ccl.soa.deploy.core\designs\com\ibm\ccl\soa\deploy\core\decoratorsemantic\package.html)

 

Plugin Contribution[Top]

 

The BADS definition will be provided as a contribution from the core plug-in with an intention to share this DS across RDM and TDM tools. The extensions of the core will contribute teh inclusions or exclusions for the content via the pattern binding.

.

 

Core BADS Extension Contribution[Top]

 

The following basic core extension needs to be contributed such as to define a BADS via the core plug-in.

<!-- Business Application Topology Decorator Semantic -->
<extension
id="com.ibm.ccl.soa.deploy.core.topology"
name="com.ibm.ccl.soa.deploy.core.topology"
point="com.ibm.ccl.soa.deploy.core.topology">

<decoratorSemantic

description="Business Application Topology is used to customize the deployment perspective for the business application usage. Use this topology type, if you are developing a topology that defines the application level components, their dependencies, hosting to conceptual nodes or define the conceptual topology with communication between nodes."


id="com.ibm.ccl.soa.deploy.core.bads"

icon="icons/bads.gif"

name="Business Application"/>

 

</extension>

<!-- *************************************************************** -->

 

 

 

 

Enabled Domains & Types [Top]

 

The BADS will enable the following domains and their defined types to be available while working with it.

Enabled Domains & Types

  • package=com.ibm.ccl.soa.deploy.core
    • Enabled Types
      • Hosting Link - com.ibm.ccl.soa.deploy.core.HostingLink
      • Dependency Link - com.ibm.ccl.soa.deploy.core.DependencyLink
      • Node
        • com.ibm.ccl.soa.deploy.core.Unit & conceptual=true
      • Component
        • com.ibm.ccl.soa.deploy.core.SoftwareComponent
  • package=com.ibm.ccl.soa.deploy.database
    • Enabled Types
      • Database Component
        • com.ibm.ccl.soa.deploy.database.DatabaseComponent
  • package=com.ibm.ccl.soa.deploy.j2ee
    • Enabled Types
      • Web Component
        • com.ibm.ccl.soa.deploy.j2ee.WebComponent
