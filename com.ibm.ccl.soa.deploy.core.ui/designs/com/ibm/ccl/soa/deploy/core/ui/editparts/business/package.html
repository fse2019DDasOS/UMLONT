SOA Deploy Business Application Topology Visualization

SOA Deployment Business Application Topology Domain [Index]

 

The SOA Deployment Business Application Topology (BAT) is an extension domain of the SOA Deployment Editor which will allow Deployment Architects to abstractly layout their business components for deployment. This document will focus on the implementation of the visualization components of the modeling elements used in the BAT domain.

Service and Requirement Editing [Top]

 

The Component figure is populated along its border with its services and requirements so that each requirement/service can be individually edited and linked. These Service and Requirement nubs can be added to a component figure using a context menu or a popup toolbar. They can also be individually deleted by selecting and deleting.

 

Upon the creation of a new service or requirement, a dialog opens that allows the user to define the service/requirement interface including the technology type (EJB, Web Service, SCA, etc.). This dialog also allows a user to define the binding types which will be used when the application is deployed.

Reuse of the SOA Core SAF Framework

The SOA Assembly Framework (SAF) from the SOA Core Tools Project will be used in the development of the service/requirement dialog and implementation handler. Since the SAF was developed using a different model, it can’t be used “as-is”, however its two plugins will be added to the deploy project and adapted to work with the deploy model.
