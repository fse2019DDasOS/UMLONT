The The 'Deployment Editor' Eclipse Properties implementation and API.

Deploy Core Editor “Properties” Package API Documentation

Abstract [Index]

 

This package implements the org.eclipse.ui.views.properties.tabbed.propertySections extension point. It supplies eight sections to the tabbed Property Sheet. The sections are known by the tabs they occupy, and are called Unit, Requirements, Capabilities, Links, Configurations, Members, and Artifacts.

This package provides a static class PropertyUtils which includes static methods for typical Property Sheet UI tasks such as supplying display names for Units and Capabilities.

Property Sheet implementation [Top]

 

In Eclipse, the Property View is made up of Categories. Each Category is made up of Tabs, and each Tab is made up of Sections.

The SOA Deploy UI plug-in contributes two categories, Unit and Appearance (but Appearance is not used). Third parties wishing to add tabs must do so using one of these categories.

The SOA Deploy UI plug-in contributes tabs property.tab.AppearancePropertySection, property.tab.UnitsPropertySection, property.tab.RequirementsPropertyTab, property.tab.RequirementsPropertyTab2, property.tab.CapabilitiesPropertyTab, property.tab.ProblemsPropertyTab, property.tab.ConfigurationsPropertyTab, property.tab.MembersPropertyTab, property.tab.LinksPropertyTab, property.tab.LinkPropertyTab, property.tab.ModulePropertySection. SOA Deploy fills (or plans to fill) each tab with a Section that will Use Extra Space so it is not recommended for third parties to add sections to existing tabs. Third parties should define their own <propertyTab ...> within the existing categories.

SOA Deploy's properties utility class is PropertyUtils.

SOA Deploy makes extensive use of <propertySection filter=...>, and the particular tabs displayed depend upon characters of the Unit selected. Third parties wishing to write a section that appears only when a Unit is selected may use the UnitEditPartPropertySectionFilter filter.
