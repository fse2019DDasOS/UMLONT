BackGround [Top]

In Constellation Project, we need to support ASC exporter to export the topology model to executable scripts which are contained in target archive .zip file. Here the script type is ASC script, including below files: build.xml: This is a kind of orchestration file which is an Ant buildfile that defines the order of tasks to be executed and which ASC agents should execute them. This buildfile can then be executed by Ant in order to carry out the desired operations. solution.properties: This is a property file containing all tasks' required attribute value. Above two files are generated according to topology model. Besides these two files, other ASC related files are also need to be included in the archive file. They are: asc.jar: All ASC related API. ascLibrary.zip: ASC library files. In the library, we define the new tasks (Macrodef).ASC exploits Ant macros as a means of codifying operations for subsequent reuse. Those Macro Tasks accept the user-specified parameter valueand pass them to the core executable file to execute. In ASC, there are several kinds of 'macros': local macros, distributed macros and remote macros. The focus on 'local macros' implies that the underlying operations affect only the local system. ASC 'distributed macros' codify operations that may affect more than one server. Unlike 'local macros', distributed macros require manual development of the remoting aspects since more than one ASC agent may be involved. And the 'remote macros' mean that the operations can be executed on a remate system via ASCRemoteAgent. ASC remote macros can be generated automatically based on local and distributed macros definition. runAll.bat,We provide a batch file to help user to run the generated orchestration file.
